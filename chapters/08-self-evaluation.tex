\section{Self-evaluation}\label{self-evaluation}

This project was developed individually by Alex Santini. This section therefore reflects my full personal contribution.

\subsection{Alex Santini}

\textbf{Role in the project.}
As sole contributor, I covered the full lifecycle: requirements, design, implementation, testing, deployment setup, observability, and report writing.

\textbf{Main contributions.}
I implemented a decentralized P2P sensor-state system with TCP + JSON messaging, gossip-based dissemination, LWW merge, membership/failure detection, and recovery (delta/full sync). I also prepared Docker-based multi-node execution, read-only HTTP/introspection endpoints, and layered automated validation (unit, in-process integration, Docker scenarios).

\textbf{Product strengths.}
Main strengths are architectural clarity (layered node design), explicit decentralized behavior, reproducible deployment, and good observability/testing coverage for convergence and recovery scenarios.

\textbf{Weaknesses and limitations.}
The system is still a course-project prototype: no authentication/authorization/TLS, in-memory state only, and limited quantitative evaluation at larger scale.
