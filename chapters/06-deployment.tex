\section{Deployment}\label{deployment}

Deployment supports both local execution and reproducible multi-node scenarios through Docker Compose.

\subsection{Install from scratch}
Required software:
\begin{itemize}
  \item \textbf{Python 3.12} (CI-validated) and \textbf{pip};
  \item \textbf{Docker} and \textbf{Docker Compose} for multi-node/container runs;
  \item optional browser for dashboard/API inspection.
\end{itemize}

\begin{verbatim}
git clone https://github.com/AlexSantini10/distributed-sensor-hub.git
cd distributed-sensor-hub
pip install -r requirements.txt
\end{verbatim}

Expected outcome: dependencies installed and node runtime ready to start.

\subsection{Configuration}
Runtime setup is environment-based (\texttt{.env.example}). Key variables include:
\begin{itemize}
  \item identity/endpoints: \texttt{NODE\_ID}, \texttt{HOST}, \texttt{PORT}, \texttt{WEB\_API\_PORT};
  \item cluster bootstrap: \texttt{BOOTSTRAP\_PEERS};
  \item heartbeat/replication tuning: \texttt{HEARTBEAT\_INTERVAL\_MS}, \texttt{GOSSIP\_*};
  \item failure detection: \texttt{PHI\_THRESHOLD\_*};
  \item optional sensor/network simulation parameters.
\end{itemize}

\subsection{Run}
Single node (example):
\begin{verbatim}
python node.py
\end{verbatim}

Expected outcome: active TCP listener + HTTP read API (e.g. \texttt{/api/introspection}).

Multi-node with Docker Compose (examples):
\begin{verbatim}
  docker compose -f docker/docker-compose-6-nodes.yml up --build -d
  docker compose -f docker/docker-compose-12-nodes.yml up --build -d
\end{verbatim}

Expected outcome: one container per node, each with dedicated identity/ports and cluster bootstrap configuration.

As an alternative to local image build, a prebuilt image can be pulled directly from GHCR:
\begin{verbatim}
docker pull ghcr.io/alexsantini10/distributed-sensor-hub:latest
\end{verbatim}
