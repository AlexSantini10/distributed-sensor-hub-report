\section{User Guide}\label{user-guide}

This guide explains how to run the system, verify expected behavior, and collect evidence from the API and UI.

\subsection{How to use the software (instructions)}

Recommended demo setup (12 nodes), from the project root directory:

\begin{verbatim}
docker compose -f docker/docker-compose-12-nodes.yml up --build -d
\end{verbatim}

Open the dashboard on any node using:

\begin{verbatim}
http://<node-ip>:<WEB_API_PORT>/ui
\end{verbatim}

For the \texttt{docker/docker-compose-12-nodes.yml} setup, use \texttt{http://localhost:10000/ui} (node-1).

Useful API checks (example on node-1, \texttt{localhost:10000}):

\begin{verbatim}
curl http://localhost:10000/api/membership
curl http://localhost:10000/api/state
curl http://localhost:10000/api/introspection/metrics
\end{verbatim}

Stop the cluster:

\begin{verbatim}
docker compose -f docker/docker-compose-12-nodes.yml down
\end{verbatim}

\subsection{Expected outcomes}

After startup and a short convergence interval, the expected outcomes are:

\begin{itemize} 
  \item \texttt{/api/membership}: peers visible for the active topology (12 nodes in this setup);
  \item \texttt{/api/state}: non-empty replicated sensor state, similar across nodes;
  \item \texttt{/api/introspection/metrics}: non-zero gossip/replication activity;
  \item dashboard (\texttt{/ui}): topology graph, peer liveness (\texttt{alive}/\texttt{suspected}/\texttt{dead}), sensor table, and recent events rendered without extra services.
\end{itemize}

If \texttt{get\_delta\_unavailable\_total} grows continuously, the delta buffer may be too small for current load or rejoin timing.
